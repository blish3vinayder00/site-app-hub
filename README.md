# site-app-hub

## 项目简介

`site-app-hub` 是一个用于归档和发布多个独立 HTML 页面的仓库。它不针对任何特定的域名或网站，而是作为一个通用的页面集合，方便开发者或团队集中管理、测试或分发一些独立的网页内容。

本仓库的主要目的是提供一个干净、结构化的空间，用于存放那些不依赖复杂后端环境的静态页面。这些页面可以是功能演示、工具界面、临时页面、组件原型等。

## 目录说明

仓库根目录下包含以下主要文件和文件夹：

```
site-app-hub/
├── pages/          # 存放所有独立 HTML 页面的文件夹
├── assets/         # 共享资源文件（CSS、JavaScript、图片等）
├── README.md       # 本文件
└── .gitignore      # Git 忽略规则
```

- **pages/**: 每个子文件夹或文件代表一个独立的页面。建议为每个页面创建单独的文件夹，以保持其相关资源的整洁。
- **assets/**: 存放多个页面可能共用的资源，如基础样式、通用脚本或图标。页面也可以在自己的文件夹中维护私有资源。
- **.gitignore**: 定义了不需要纳入版本控制的文件或文件夹，例如本地缓存、临时文件等。

## 页面归档说明

本仓库中的每个页面都是独立、自包含的 HTML 文件。它们可以：

- 在本地浏览器中直接打开运行。
- 通过 GitHub Pages 或其他静态托管服务部署后在线访问。
- 作为项目的一部分被引用或嵌入到其他环境中。

### 页面命名规范

- 使用小写字母、数字和连字符（`-`）命名文件夹和文件。
- 每个页面的主文件统一命名为 `index.html`。
- 页面相关的其他资源（CSS、JS、图片等）建议放在同一文件夹内。

### 示例结构

```
pages/
├── example-tool/
│   ├── index.html
│   ├── style.css
│   └── script.js
├── demo-page/
│   └── index.html
└── ...
```

## 维护说明

- **添加新页面**：在 `pages/` 下创建新文件夹，放入 `index.html` 及相关资源，并确保页面可以独立工作。
- **修改现有页面**：直接编辑对应文件夹中的文件。注意不要破坏其他页面的功能。
- **删除页面**：移除对应的文件夹即可。如果页面被外部引用，请提前更新引用链接。
- **共享资源**：如果多个页面需要相同的样式或脚本，请将其放入 `assets/` 文件夹，并在页面中通过相对路径引用。

## 贡献与使用

本仓库欢迎任何形式的贡献，包括但不限于：

- 提交新的独立页面
- 改进现有页面的代码或样式
- 优化资源文件结构

如果您希望贡献，请遵循以下基本原则：

1. 确保页面独立、无外部依赖（除非依赖是共享资源）。
2. 页面不应包含任何恶意、侵权或违反平台政策的内容。
3. 提交前请测试页面在主流浏览器中的基本兼容性。
